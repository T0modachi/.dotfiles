# AGENTS.md — Guide for coding agents

> This file was built from the CodeGraph index (`codegraph.db`) of this repository. Use it as the source of conventions, structure, and caveats before modifying these dotfiles.

## 1. Project summary

This repository is a **NixOS + Home Manager** configuration for one machine and one user. Everything is declared as a Nix flake (`flake.nix`) and manages both the operating system and the user environment.

- **Fixed system:** `x86_64-linux`
- **Nixpkgs channel:** `nixos-unstable`
- **Home Manager:** `master` branch with `follows = "nixpkgs"`
- **Key flake inputs:** `nixpkgs`, `home-manager`, `devenv`, `nixvim`, `llm-agents`, `mcp-servers-nix`, `agent-skills-nix`, `zen-browser-flake`, `caveman`, `anthropic-skills`, `ponytail`

## 3. Flake outputs

`flake.nix` exposes two kinds of outputs:

### NixOS configurations

- `nix-laptop` — NixOS laptop. Uses `./system` (`default.nix`) as the root module.

### Home Manager configurations

- `T0modachi` — User environment. Uses `./home` (`default.nix`) as the root module.

## 4. Editing conventions

- **Nix is functional:** do not mutate variables or rely on impurity. All repo resource paths must resolve to the Nix store via `source = ../../relative/path`.
- **Relative paths:** in `home/dotfiles.nix` use `../configs/<app>/` to reference user configurations.
- **Package configs belong under `configs/<app>/`**, not at the repository root.
- **Do not hardcode hashes when avoidable:** Zen Browser is now consumed from the community flake (`zen-browser-flake`) instead of a pinned AppImage hash.
- **Language:** write all agent-authored documentation and new comments in **English**. Do not remove existing Spanish comments without a clear reason.
- **Allow unfree:** centralized in `system/nix-settings.nix` (`nixpkgs.config`). Do not duplicate it.
- **Permitted insecure packages:** `electron-25.9.0`, `electron-27.3.11`, `electron-28.3.3`, `electron-39.8.10`. Do not remove them while they are still required.

## 7. Activation scripts

`apply.sh` unifies activation:

```bash
./apply.sh          # Apply system + home (default)
./apply.sh system   # Only nixos-rebuild switch
./apply.sh home     # Only home-manager activate
```

Other helper scripts:

- `update.sh` — `nix flake update`
- `gc.sh` — `nix-collect-garbage --delete-older-than 30d`

## 9. Caveats and risks

- **`system/hardware.nix`** is auto-generated by `nixos-generate-config`. Do not edit it by hand unless the hardware changes.
- **`.codegraph/`** is git-ignored. Do not commit the database.
- **`result`** is a Nix build symlink. It is git-ignored.
- **Do not duplicate `allowUnfree`** — it already lives in `system/nix-settings.nix`.
- **`system/packages.nix`** is only for base system tools. User apps and tools belong in `home/packages.nix`.

## 10. How to verify a change

1. **Evaluate:** `nix flake check`
2. **Format Nix:** `nixfmt` or `alejandra` if available.
3. **Apply:** `./apply.sh [system|home|all]`
4. **Reindex CodeGraph:** `codegraph sync .`

## 11. Quick rules for agents

- **Add user packages to `home/packages.nix`, not `system/packages.nix`.**
- **Do not edit `system/hardware.nix`.**
- **Do not add flake inputs without discussion;** each input increases evaluation and lock time.
- **Prefer `home.file` with Nix-store-relative paths** over `.text` templates with `builtins.readFile`.
- **After editing, run `codegraph sync .`** to keep the index current.
- **When creating or deleting files or directories, run `codegraph sync .` and update `AGENTS.md`.**
- **New modules:** one file per responsibility. If a module grows beyond ~50 lines, consider splitting it.
